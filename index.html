用Construct3制作平台跳跃游戏

2024-09-14
游戏设计作业

1. 前言

Construct 3是一款基于 HTML5 的 2D 游戏开发工具，它免于编程的特点能够帮助不会写代码的人构建游戏。它能够在浏览器中运行，实时调试修改，并且能导出到网页、桌面、移动设备等多种平台。对于游戏制作初学者而言，它是一个理想而友好的入门工具。

本博客记录了笔者初次使用Construct 3构建简单的平台跳跃游戏的思考与实践过程。

2.2 规则

  • 玩家操控的角色是一只狗。狗可以通过方向键操控，连续按两次↑能够二段跳，按住↓会趴下。趴下时可以移动
  • 狗踩到浅色冰块时，冰块将破碎
  • 雪人被碰撞到后，雪人将融化
  • 狗碰撞骨头时，狗会开始吃骨头。狗吃骨头时不受玩家控制
  • 狗碰到蓝色星星时，星星会消失
  • 狗与蘑菇重叠时，蘑菇会发光
  • 两个雪屋可以相互传送
  • 狗掉落到底部虚空后会重新回到出生点
  • 玩家胜利条件是：收集所有星星！吃掉所有骨头！融化所有雪人！
  • 游戏所有贴图是从独立游戏《undertale》中拆包得到，所有权属于游戏作者tobyfox，不属于我！

3. 制作心得

3.1 移动

  • Construct3（下文简称C3）预设了最基本的角色移动控件。左右转、跳跃、二段跳、平台识别、重力、加速度等物理相关功能几乎一键搞定，十分方便。
  • 关键点是把主控角色的【行为】模块中添加【平台】与【镜头跟随】
  • 我给角色定义了两个动画。正在移动时播放walk，准备停下和准备落地时播放still。

3.2 角色死亡与重开

  • 由于C3场景大小有限，我很自然地将【角色离开场景】定位角色死亡-重开的判定标志。下坠死亡符合一般逻辑，但调试过程中我发现在一些位于场景偏上方的平台上二段跳时，角色可能会超出画布高度，此时也会被系统判定为死亡。这有些反直觉了。所以我将画面的上方和左右两边都添加了边框。
  • 角色离开场景时，销毁角色，重载场景&重置全局变量（这个容易忘记！）。角色初次出生与重生时，会播放一个【狗被绳子从画面顶端吊下来】的动画。

3.3 事件表

  • 非常好事件表，让我不用敲代码

  • 值得注意的是，事件表的条件相当于编程中的while。我一开始一直以为它相当于if，导致出现了很多问题。

    • 比如，我想实现一个效果：当狗碰撞到骨头时，【狗】开始播放【吃骨头】的动画，而【骨头】被销毁。按照【条件：狗碰撞骨头；动作：播放动画+销毁骨头】这个逻辑写出来后，发现骨头确实没了，但吃骨头的动画也消失了。这是因为系统判定狗与骨头接触的时候才会播放动画，不接触时直接停止动画，而两个动作是没有先后顺序的，碰到骨头的一瞬间销毁骨头，那动画自然也不会播放了。

  • 雪人融化、碎冰冰和吃星星都挺简单的没什么好写的，碰撞后切一个动画就行

  • 雪屋传送

    • 逻辑是碰撞后把角色位置设置到另一个坐标。注意另一个坐标位置不能和雪屋重叠，不然就无限循环传送了

3.3 关卡设计

  • 因为只做了一个图，所以严格意义上并没有关卡，但我确实为了游玩体验花了一点小心思
  • 确保了玩家不会被困在任何一个平台上，但也制作了一些距离比较极限的路径！
  • 在距离出生点很近的浅色冰块上放置了星星，引导玩家踩上去，提示玩家该冰块会破裂
  • 有两处地方需要玩家按下↓键，使狗趴下，减少角色高度，才能钻进缝隙吃到星星。在这两处地方的入口都放置了一块骨头（狗吃骨头时会趴下，算是一个小提示）
